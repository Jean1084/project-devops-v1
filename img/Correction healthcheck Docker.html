ChatGPT a dit :

Le problème vient du fait que le conteneur simple-api-jean n'a pas de commande de test pour le healthcheck. Docker ne peut donc pas vérifier son état, ce qui empêche depends_on de fonctionner correctement.

✅ Correctif : Ajouter une commande au healthcheck

Ajoutez test: ["CMD", "curl", "-f", "http://localhost:5000"] dans simple-api-jean pour que Docker puisse vérifier si l'API est bien accessible.

🔧 Fichier corrigé :

yaml
services:
  simple-api-jean:
    image: simple-api-jean:latest
    ports:
      - "4000:5000"
    volumes:
      - ./simple_api/student_age.json:/data/student_age.json
    networks:
      - simple-api-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000"]  # Vérifie si l'API répond
      interval: 10s
      retries: 5
      start_period: 20s
  web-php-apache-jean:
    image: php-api-jean:latest
    depends_on:
      simple-api-jean:
        condition: service_healthy
    ports:
      - "8082:80"
    volumes:
      - ./website/www:/var/www/html
    environment:
      USERNAME: jean
      PASSWORD: agree
    networks:
      - simple-api-network

networks:
  simple-api-network:
    driver: bridge

🔥 Explication :

  1. Ajout d'un test de santé (healthcheck) :

    • Utilise curl pour vérifier si simple-api-jean répond bien sur http://localhost:5000.

    • -f fait en sorte que curl échoue si la réponse HTTP n'est pas correcte.

  2. Pourquoi ?

    • Sans test, Docker ne sait pas comment vérifier l'état du conteneur.

    • depends_on: condition: service_healthy ne peut être utilisé que si un healthcheck valide est défini.

🛠 Commandes à exécuter après modification :

bash
docker-compose down
docker-compose up -d --build

Cela permettra de recharger la configuration et de reconstruire les images si nécessaire. 🚀
